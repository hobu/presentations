USACE

  • Contacted me in about the libLAS project
  • Wanted to store point cloud data in Oracle
  • Tons of data related to Glaciology
  • Need for data management system beyond Fedex

GRiD

  • Pilot project in 2009
  • Fielded operational GRiD system in Jan 2013 for NGA
  • Holdings growing by 1000s of files per day
  • 350-500 tb of point cloud data

PDAL

  • GDAL's point cloud distant cousin
  • Focused on translation and filtering
  • Processing pipeline and format translation for LiDAR
  • Complimentary to libraries such has PCL
  • Geo-centric rather than vision- or vizualization-centric

PDAL's support of GRiD

  • Data chipping
  • Consume wide range of formats and store into Oracle
  • Manage and preserve metadata into a common schema
  • User-directed export and subsetting
  • Server-based exploitation activities
